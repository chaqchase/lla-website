# NPM Plugin

Search and explore NPM packages directly from your terminal with integrated bundle size analysis.

## Features

- **NPM Package Search** - Find packages from the NPM registry
- **Bundlephobia Integration** - Real bundle size information (minified & gzipped)
- **Package Details** - Version, downloads, description, and metadata
- **Bundle Size Analysis** - See actual package sizes before installing
- **Favorites Management** - Save frequently used packages
- **Browser Integration** - Open package pages directly
- **Search History** - Track your package searches
- **Interactive Interface** - Beautiful terminal UI

## Installation

The npm plugin is included with lla's first-party plugins:

```bash
lla install
```

Or enable it if already installed:

```bash
lla use  # Interactive plugin manager
```

## Usage

### Search Packages

Launch interactive package search:

```bash
lla plugin npm search
```

You'll be prompted to:
1. Enter a package name
2. View package details with bundle size
3. Choose an action:
   - Copy install command to clipboard
   - Add to favorites
   - Open npm page in browser

The search displays:
- Package name and latest version
- Description and author
- License and homepage
- Bundle size (minified + gzipped from Bundlephobia)

### Manage Favorites

View and manage your favorite packages:

```bash
lla plugin npm favorites
```

Interactive menu lets you:
- View package details for any favorite
- Copy install commands (single or multiple)
- Remove packages from favorites

### Configure Preferences

Set your preferred package manager:

```bash
lla plugin npm preferences
```

Choose from:
- npm (default)
- yarn
- pnpm
- bun

This affects the install commands shown and copied to clipboard.

## Positional Syntax

Use the shorter positional syntax (v0.5.0+):

Create a short alias for faster access:

```toml

```

Then use:

```bash

```

Usage:


## Bundle Size Information

### What is Bundle Size?

The plugin shows two important metrics:

- **Minified**: Size after removing whitespace and shortening variable names
- **Gzipped**: Compressed size (actual transfer size over the network)

### Understanding Sizes

```bash
lla plugin npm details "react"

# Output shows:
# Bundle Size:
#   Minified: 130.5 KB
#   Gzipped:  41.2 KB
```

**Interpretation:**
- Small (<50 KB gzipped): ✅ Great
- Medium (50-200 KB gzipped): ⚠️ Watch out
- Large (>200 KB gzipped): ❌ Consider alternatives

### Size Comparison

Compare bundle sizes when choosing packages:

```bash
# Compare date libraries
lla plugin npm details "moment"      # ~71 KB gzipped
lla plugin npm details "date-fns"    # ~12 KB gzipped
lla plugin npm details "dayjs"       # ~2 KB gzipped
```

## Interactive Features

### Search Actions

After searching for a package, you can:
- **Copy Install Command** - Copies the package manager-specific install command to clipboard
- **Add to Favorites** - Saves the package for quick access later
- **Open NPM Page** - Opens npmjs.com in your default browser

### Favorites Actions

When viewing favorites, you can:
- **View Package Details** - See full package info with latest bundle size
- **Copy Install Commands** - Select multiple packages and copy all install commands at once
- **Remove from Favorites** - Multi-select packages to remove

## Troubleshooting

### No Results Found

If search returns nothing:
1. Check package name spelling
2. Package may be deprecated
3. Check NPM registry status
4. Verify internet connection

### Size Information Missing

If bundle size doesn't show:
1. Package may not be on Bundlephobia
2. Package might not be JavaScript/browser-compatible
3. Too large to analyze
4. Network connectivity issue

### Favorites Not Saving

If favorites don't persist:
1. Check config directory permissions
2. Verify lla config is initialized
3. Run `lla init` if needed

### Clipboard Not Working

If copy to clipboard fails:
1. Verify clipboard utilities are installed
2. macOS: Should work out of the box
3. Linux: Install `xclip` or `xsel`
4. Windows: Should work out of the box
