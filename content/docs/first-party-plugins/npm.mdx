# NPM Plugin

Search and explore NPM packages directly from your terminal with integrated bundle size analysis.

## Features

- **NPM Package Search** - Find packages from the NPM registry
- **Bundlephobia Integration** - Real bundle size information (minified & gzipped)
- **Package Details** - Version, downloads, description, and metadata
- **Bundle Size Analysis** - See actual package sizes before installing
- **Favorites Management** - Save frequently used packages
- **Browser Integration** - Open package pages directly
- **Search History** - Track your package searches
- **Interactive Interface** - Beautiful terminal UI

## Installation

The npm plugin is included with lla's first-party plugins:

```bash
lla install
```

Or enable it if already installed:

```bash
lla use  # Interactive plugin manager
```

## Usage

### Search Packages

Search the NPM registry:

```bash
lla plugin npm search "react"
```

Results show:
- Package name
- Latest version
- Weekly downloads
- Description
- Bundle size (minified + gzipped)

### View Package Details

Get detailed information about a specific package:

```bash
lla plugin npm details "lodash"
```

Displays:
- Full package metadata
- All available versions
- Dependencies
- Repository information
- Bundle size breakdown
- Download statistics

### Manage Favorites

Add packages you use frequently:

```bash
# Add to favorites
lla plugin npm add-favorite "express"

# List favorites
lla plugin npm list-favorites

# Remove from favorites
lla plugin npm remove-favorite "express"
```

### Search History

View and manage your search history:

```bash
# View history
lla plugin npm history

# Clear history
lla plugin npm clear-history
```

### Open in Browser

Open the package page on npmjs.com:

```bash
lla plugin npm open "typescript"
```

## Positional Syntax

Use the shorter positional syntax (v0.5.0+):

```bash
# Search
lla plugin npm search "axios"

# Package details
lla plugin npm details "webpack"

# Manage favorites
lla plugin npm add-favorite "vite"
lla plugin npm list-favorites
```

## Plugin Aliases

Create a short alias for faster access:

```toml
# ~/.config/lla/config.toml
[plugin_aliases]
n = "npm"
```

Then use:

```bash
lla plugin n search "react"
lla plugin n details "vue"
```

## Shortcuts

Create shortcuts for common NPM operations:


Usage:


## Bundle Size Information

### What is Bundle Size?

The plugin shows two important metrics:

- **Minified**: Size after removing whitespace and shortening variable names
- **Gzipped**: Compressed size (actual transfer size over the network)

### Understanding Sizes

```bash
lla plugin npm details "react"

# Output shows:
# Bundle Size:
#   Minified: 130.5 KB
#   Gzipped:  41.2 KB
```

**Interpretation:**
- Small (<50 KB gzipped): ✅ Great
- Medium (50-200 KB gzipped): ⚠️ Watch out
- Large (>200 KB gzipped): ❌ Consider alternatives

### Size Comparison

Compare bundle sizes when choosing packages:

```bash
# Compare date libraries
lla plugin npm details "moment"      # ~71 KB gzipped
lla plugin npm details "date-fns"    # ~12 KB gzipped
lla plugin npm details "dayjs"       # ~2 KB gzipped
```

## Troubleshooting

### No Results Found

If search returns nothing:
1. Check package name spelling
2. Try broader search terms
3. Package may be deprecated
4. Check NPM registry status

### Size Information Missing

If bundle size doesn't show:
1. Package may not be on Bundlephobia
2. Package might not be JavaScript
3. Too large to analyze
4. Network connectivity issue

### Favorites Not Saving

If favorites don't persist:
1. Check config directory permissions
2. Verify lla config is initialized
3. Run `lla init` if needed

## Best Practices

### 1. Check Sizes First

Always review bundle size before installing:
```bash
lla plugin npm details "package-name"
```

### 2. Compare Alternatives

Evaluate multiple options:
```bash
lla npms "state management"
# Compare redux, zustand, jotai, etc.
```

### 3. Use Favorites

Track your standard packages:
```bash
lla plugin npm add-favorite "your-go-to-packages"
```

### 4. Monitor Bundle Impact

Be aware of cumulative size:
```bash
# Each package adds up
lla npmi "package1"  # +50 KB
lla npmi "package2"  # +30 KB
lla npmi "package3"  # +40 KB
# Total: +120 KB to your bundle
```

### 5. Review History

Learn from past research:
```bash
lla plugin npm history
```
